Welcome to Git

Overview of Git and GitHub

Git is a free, open source and very commonly used version control system. Git is known to be used to track your source code and many softwares project rely on Git thanks to it version control. It is widely recommended since it allows many to work together using the same files, and it also stores software development history in a reposistory.

GitHub is a code hosting platform made by Microsoft for version control and collaboration using Git. GitHub helps developers to store, manage as well as tracking and making changes to their codes. GitHub is also one of the largest coding communities that let you save your codes online and allow developers of a project to see changes one has made, and it is one of the easiest ways to get your projects known.

How to install Git and GitHub

Windows

1. Go to Git Installer page to download the latest version of Git for Windows. 2. Follow the instructions in the Setup Wizard. 3. Open the windows command promt (or Git Bash). 4. Type 'git version' to verify that Git was installed.

Mac

1. Go to Git Installer page to download the latest version of Git for MacOS. 2. Follow the instructions provided. 3. Open the command promt 'terminal'. 4. Type 'git version' to verify that Git was installed.

Basic Git commands

• 'pull': Used to download content from remote reposistories. Ex.: 'git pull' • 'add': This command adds a change in the directory to the staging area. Ex.: 'git add *' • 'status': Displays the state of the working directory and staging area. Ex.: 'git status' • 'commit': Used to save changes. Ex.: 'git commit -m [commit]' • 'push': Opposite to 'pull', 'push' is used to upload content to remote reposistories. Ex.: 'git push'

How to resolve conflicts

Branching

A branch is a way to code a new feature or modification to the software and still not affecting the main part of the project.

Creating a branch: start on a default branch or create a new branch.

• Create a new branch based on the current HEAD: 'git branch [new-branch]' • Create a new branch based on an existing branch: 'git branch [new-branch] [base-branch] • Create a new branch from a remote branch: 'git branch --track [new-branch] origin/[base-branch] • Create a new branch from a remote reposistory: 'git push -u origin [local-branch]

Merging a branch: a running branch can merge with any other branch in your Git repository: 'git merge [commit]

Deleting a branch: a running branch can be deleted from the reposistory.

• Delete a branch locally: 'git branch -d [branch]' • Delete a branch remotely: 'git push [remote] --delete [branch]'

Checking out a branch: a running branch can pull or checkout to make a clone of it so that the user can work on any of them.

• Create a new branch and switch to it: 'git check out -b' • Switch between branches or checking out a remote branch: 'git checkout'
